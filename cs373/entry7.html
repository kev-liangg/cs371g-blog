Entry #7

  • March 7, 2021
  • Kevin Liang
What did you do this past week?

This past week, I put in a lot of work on Phase I of the IDB project along with my group. I specifically implemented the dynamic about page and the stylized landing page, and worked to link our AWS cloudfront distribution to the Namecheap domain name. Everything ended up working out in the end, but it was definitely submitted way too close to the deadline. We will definitely have to be more proactive with time management as the difficulty of the project ramps up.

What's in your way?

Nothing in particular is in my way, now that my team has cleared the major phase deadline. We were able to get a headstart by implementing the website already with a React frontend and Python backend and verifying that those successfully deploy. We spent a lot of time learning React and Javascript, and now it looks like we might have to do the same learning from scratch with Python/wsgi and SQL.

What will you do next week?

Next week, I will definitely jump into Phase II with the rest of my team. Even with our backend setup, it seems like we will have to spend a lot more time implementing our SQL designs and working with the actual data.

If you read it, what did you think of the Open-Closed Principle?

I really enjoyed reading the Open-Closed Principle paper. A lot of what it covered was either familiar or intuitive to me, and it was a great validation to how far I’ve come as a software developer. I will always think there’s a certain beauty to object-oriented design, and I’m glad to be applying this again through Javascript and React in some ways.

What was your experience of iterators and reduce2?

I already had a very solid understanding of iterators since I took Generic Programming with Dr. Downing last summer. Iterators are irreplaceable there, since a lot of generic templated functions will end up taking in an iterator as a parameter, and the function needs to work with the “weakest” possible iterator. A lot of it is definitely outside the scope of our Python-based lectures, but it was very reassuring seeing the concept emerge again. The exercise reduce2 was straightforward, but apparently my team came up with a non-intended approach. We didn’t end up explicitly throwing the exception in one case, but ended up using the underlying Python machinery to throw the exception for us which somehow still managed to pass that test case.

What made you happy this week?

I talked to my father about possibly starting therapy for my constant self-hatred issues. He was supportive about it, which is good, but I’m still a bit unsure if I should take the chance. I’m covered under his insurance plan but the deductible is really high. I’m not particularly keen to pay so much out of pocket for something that’s not guaranteed to fix anything.

What's your pick-of-the-week or tip-of-the-week?

This interactive guide was very helpful for setting up multiple pages in the React frontend that will be required for Phase II. I remember being really confused since it was so different from simply linking HTML pages together, but having the code sandbox was really helpful for understanding what every part of the code did. It becomes almost trivial past the initial learning curve, and this resource will definitely help reach that point.
